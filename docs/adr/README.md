# Architecture decision records

An ADR records a decision that affects more than one module, is expensive to reverse, or establishes a platform constraint. ADRs are immutable after acceptance; a later ADR supersedes an earlier one.

## Template

```md
# ADR-NNN: Title

Status: Proposed | Accepted | Superseded
Date: YYYY-MM-DD

## Context

## Decision

## Consequences

## Alternatives considered

## References
```
