# Product roadmap

This is an ordered capability roadmap, not a date commitment. Work should advance only when its prerequisites and quality gates are met.

## V1 ? trustworthy campus utility

- Stabilize provider data contracts and provenance.
- Complete the building/faculty/dining/parking discovery loop.
- Make map search, selection, directions, and fallback states reliable.
- Establish accessibility, error, loading, and deployment baselines.
- Add observability for critical student tasks.

## V2 ? unified discovery and personalization

- Consolidate search ranking across entity types.
- Add explicit favorites and recently viewed entities backed by authenticated storage.
- Return structured Lumi answers with citations and action cards.
- Introduce provider onboarding documentation and validation tooling.

## V3 ? multi-campus platform

- Scope persisted data by university and campus.
- Move ingestion to repeatable, auditable provider pipelines.
- Introduce a provider registry and a second provider as a contract test.
- Add an administrator workflow for content corrections and source review.

## Deferred until foundation is proven

Indoor navigation, live occupancy, real-time shuttle feeds, AR, and deep LMS integrations require verified source agreements, operations support, and robust privacy/security designs.
