# UX principles

1. **Start from a student task.** ?Find parking near Rawls? is a better design prompt than ?build a parking page.?
2. **Give every state a next step.** Empty, error, offline, and unavailable states suggest a recovery route.
3. **Keep context visible.** A result should preserve campus, data freshness, and location context.
4. **Favor direct manipulation.** Selecting an entity should synchronize map, details, and available actions without making users repeat a search.
5. **Do not fake certainty.** Communicate ?reported,? ?estimated,? or ?last verified? when information is not live.
6. **Reduce navigation cost.** Search, recent items, and clear deep links should reach common destinations quickly.
7. **Design responsive flows, not squeezed layouts.** Mobile maps use bottom sheets and thumb-friendly actions; desktop can use side panels and keyboard shortcuts.
